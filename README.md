# FP-Josephus

Julia implementations of the algorithms for the **Josephus function** `J_k` and its
**fixed points** developed in the series of papers by J. Y. Bello-Cruz and
R. Quintero-Contreras.

In the Josephus problem, `n` people stand in a circle and every `k`-th person is
removed until one survivor remains; `J_k(n)` is the survivor's starting position. A
*fixed point* is an `n` with `J_k(n) = n` — the survivor ends up exactly where the
counting began. This package generates fixed points of `J_k` for any `k >= 2` and
evaluates `J_k(n)` efficiently, using the structural recurrences from the four papers
listed below rather than the slow `O(n)` definition.

## Installation

From the Julia REPL:

```julia
import Pkg
Pkg.develop(url="https://github.com/<your-account>/FP-Josephus")
using FPJosephus
```

Or clone the repository and activate the project:

```bash
git clone https://github.com/<your-account>/FP-Josephus
cd FP-Josephus
julia --project=. -e 'using Pkg; Pkg.instantiate()'
```

## Quick start

```julia
using FPJosephus

# Survivor of the classical instance: 41 people, every 3rd removed.
josephus(3, 41)                # 31

# First eight fixed points of J_4.
fixed_points_J4(8)             # [1, 21, 38, 51, 122, 163, 689, 919]

# Fixed points of J_k for any k.
fixed_points(7, 6)             # [1, 2, 3, 12, 68, 274]

# Evaluate J_4 at a large argument without scanning to it.
josephus4_evaluate(1_000_000)  # 333542
```

## What each paper contributes

| Paper | Scope | Functions |
|-------|-------|-----------|
| **[1]** Analytical Study and Efficient Evaluation of the Josephus Function | general `k`: Euler's formula, extremal-point theory, fast evaluation | `josephus`, `josephus_euler`, `josephus_knuth`, `josephus_extremal`, `next_high_extremal`, `high_extremal_points`, `fixed_points` |
| **[2]** On the Recurrence Formula for Fixed Points of the Josephus Function | `k = 3`: closed recurrence for the fixed points | `fixed_points_J3`, `fixed_points_J3_recurrence`, `num_pure_points_J3`, `josephus3_via_fixed_points` |
| **[3]** Fixed Points of the Josephus Function via Fractional Base Expansions | `k = 3`: Chinese Remainder Theorem and base-3/2 expansions | `fixed_point_J3_crt`, `base_three_halves` |
| **[4]** A 3-adic Recurrence for the Fixed Points of the Josephus Function `J_4` | `k = 4`: block structure and 3-adic recurrence | `fixed_points_J4`, `next_fixed_point_J4`, `josephus4_evaluate`, `block_structure_J4`, `first_block_J4` |

## A tour of the API

### General `k` (paper [1])

The Josephus function itself, by three methods that all return the same value:

```julia
josephus_euler(5, 1000)     # exact O(n) recurrence (the reference)
josephus_knuth(5, 1000)     # Knuth's O(k log n) method
josephus_extremal(5, 1000)  # the extremal algorithm of [1], O(log n) jumps
```

The extremal algorithm exploits that `J_k` is piecewise linear of slope `k` between
*high extremal points*. You can inspect those points directly:

```julia
high_extremal_points(4, 200)    # (n_e, J_4(n_e)) pairs up to 200
```

Fixed points for any `k` are generated by walking the high extremal points and keeping
those that land on the diagonal:

```julia
fixed_points(10, 6)   # [1, 4, 475, 8177, 11217, 28954]
```

### `k = 3` (papers [2] and [3])

For `k = 3` the fixed points obey a clean recurrence driven by a single 2-adic
valuation: the number of pure high extremal points between consecutive fixed points
`n_p` and the next is `ν_2(3 n_p + 2)`, and

```
n_p_next = (3^m (3 n_p + 2) - 2^m) / 2^(m+1),   m = ν_2(3 n_p + 2).
```

```julia
fixed_points_J3(13)
# [1, 2, 13, 20, 46, 157, 236, 532, 1198, 4045, 6068, 13654, 46084]

num_pure_points_J3(46)            # ν_2(3*46+2) = ν_2(140) = 2
josephus3_via_fixed_points(10_000)  # evaluate J_3 using only fixed points
```

Paper [3] characterizes each fixed point through the Chinese Remainder Theorem and a
base-3/2 expansion:

```julia
base_three_halves(13)   # [2, 1, 2, 1]   (digits in {0,1,2}, value Σ d_i (3/2)^i)

# a fixed point x = n_p^{(l+1)} satisfies 2x+1 ≡ 0 (mod 3^p), 3x+2 ≡ 0 (mod 2^q)
fixed_point_J3_crt(20, 1, 2)   # true
```

### `k = 4` (paper [4])

For `k = 4` the pure points come in two types and group into alternating blocks; the
block lengths come from 3-adic valuations. The package generates the fixed points,
exposes the block structure, and evaluates `J_4` from it.

```julia
fixed_points_J4(8)        # [1, 21, 38, 51, 122, 163, 689, 919]
next_fixed_point_J4(5167) # 51617

# The interval (5167, 51617) splits into five blocks, listed as
# (type, length, last point):
block_structure_J4(5167)
# [(2, 1, 6889), (1, 1, 9186), (2, 3, 21775), (1, 1, 29034), (2, 1, 38712)]

josephus4_evaluate(10_000)  # 3254
```

## Running the tests

```bash
julia --project=. -e 'using Pkg; Pkg.test()'
```

The test suite checks every fast method against the exact Euler recurrence over tens
of thousands of inputs, and checks the fixed-point generators against a brute-force
scan for `k = 2, ..., 15`.

## Examples

Two runnable scripts are in [`examples/`](examples):

```bash
julia --project=. examples/generate_fixed_points.jl
julia --project=. examples/solve_josephus.jl
```
